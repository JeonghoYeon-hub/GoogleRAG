# 파일 RAG 검색 시스템

Google Gemini AI를 활용하여 업로드한 문서에 자연어로 질문하고 답변을 받을 수 있는 로컬 실행형 RAG(검색 증강 생성) 시스템입니다. PDF, Word, Excel, PowerPoint, Markdown 등 다양한 파일을 업로드하고, Google Drive 및 Notion과 연동하여 문서를 바로 가져올 수 있습니다.

---

## 시스템 요구사항

| 항목 | 최소 사양 |
|------|-----------|
| 운영체제 | Windows 10/11, macOS 12+, Ubuntu 20.04+ |
| Python | 3.10 이상 |
| 인터넷 | 필수 (Gemini API 호출) |
| 브라우저 | Chrome, Edge, Firefox 최신 버전 |

---

## 설치 방법

### Windows 빠른 시작 (권장)

1. Python 3.10 이상을 설치합니다.  
   [python.org](https://www.python.org/downloads/) 에서 다운로드 — 설치 시 **"Add Python to PATH"** 반드시 체크
2. 프로젝트 폴더 전체를 원하는 위치에 압축 해제합니다.
3. **`시작.bat`** 파일을 더블클릭합니다.
4. 최초 실행 시 필요한 패키지가 자동으로 설치됩니다 (5~10분 소요).
5. 설치 완료 후 서버가 자동 실행됩니다.
6. 브라우저에서 `http://localhost:8000` 을 엽니다.

> 서버를 종료하려면 터미널 창을 닫으면 됩니다.

---

### 수동 설치 (Windows / macOS / Linux 공통)

```bash
# 1. 프로젝트 폴더로 이동
cd 파일RAG검색시스템

# 2. 가상환경 생성 (권장)
python -m venv venv

# Windows
venv\Scripts\activate
# macOS / Linux
source venv/bin/activate

# 3. 패키지 설치
pip install -r requirements.txt

# 4. 서버 실행
python app.py
```

5. 브라우저에서 `http://localhost:8000` 접속

## API 키 설정

### Gemini API Key (필수)

Gemini AI로 파일 검색과 질의응답을 수행합니다.

1. [Google AI Studio](https://aistudio.google.com/apikey) 접속 후 Google 계정으로 로그인
2. **"Create API key"** 클릭
3. 생성된 키 (`AIza...` 로 시작) 복사
4. 앱 설정 화면의 **Gemini API Key** 입력란에 붙여넣기 → **저장**

> Gemini 2.5 Flash 및 File Search Store API를 사용합니다. 무료 티어로 테스트 가능하나 대용량 사용 시 요금이 발생할 수 있습니다.

---

### Notion API Key (선택)

Notion 페이지와 데이터베이스를 직접 불러와 업로드할 수 있습니다.

**키 발급:**

1. [Notion 내 통합](https://www.notion.so/my-integrations) 접속
2. **"+ 새 통합 만들기"** 클릭
3. 이름 입력 (예: `파일RAG검색`) → **"제출"**
4. **"내부 통합 시크릿"** (`secret_...` 로 시작) 복사
5. 앱 설정에서 **Notion API Key** 항목에 붙여넣기 → **저장**

**Notion 페이지 연결:**

가져오려는 각 Notion 페이지/데이터베이스를 연결해야 합니다.

1. Notion에서 연결할 페이지 열기
2. 우측 상단 `···` 클릭 → **연결** → 위에서 만든 통합 선택

> 통합에 연결된 페이지만 앱에서 검색·선택할 수 있습니다.

---

### Google Drive (선택)

Drive에 있는 파일을 직접 선택하여 업로드할 수 있습니다.  
Drive 연동을 사용하려면 관리자가 미리 `client_secret.json` 파일을 준비해야 합니다.

#### client_secret.json 발급 절차

1. [Google Cloud Console](https://console.cloud.google.com/) 접속 → 프로젝트 선택 또는 생성
2. **API 및 서비스 → 라이브러리** 에서 "Google Drive API" 검색 → **사용 설정**
3. **API 및 서비스 → 사용자 인증 정보 → + 사용자 인증 정보 만들기 → OAuth 클라이언트 ID**
4. 애플리케이션 유형: **웹 애플리케이션** 선택
5. **승인된 리디렉션 URI** 에 아래 주소 추가:
   ```
   http://localhost:8000/auth/callback
   ```
6. 생성 후 **JSON 다운로드** → 파일명을 `client_secret.json` 으로 변경
7. 프로젝트 폴더 (`app.py` 와 같은 위치)에 저장

#### OAuth 동의 화면 설정

1. **API 및 서비스 → OAuth 동의 화면** 이동
2. 게시 상태가 **테스트 중** 이면 → **테스트 사용자** 에 연결할 Google 계정 추가
3. 스코프에 `drive.readonly` 포함 확인

#### 사용자 연결 방법

1. 앱에서 열쇠 아이콘 클릭 → **Google Drive → Google 계정 연결** 버튼 클릭
2. 팝업이 열리면 연결할 Google 계정으로 로그인
3. 권한 허용 클릭 → 팝업이 자동으로 닫히고 연결 완료

> 브라우저별로 각자 Google Drive 계정을 연결할 수 있습니다. 같은 서버에 여러 명이 접속해도 각자의 Drive에 개별 접근합니다.

## 사용 방법

### 1단계: 서버(파일스토어) 만들기

**서버**는 파일을 묶어두는 그릇입니다. 프로젝트·주제·부서별로 여러 개를 만들 수 있습니다.

1. 화면 왼쪽 패널 상단 **"+"** 버튼 클릭
2. 서버 이름 입력 (예: `회사 정책 문서`, `2024 프로젝트 자료`)
3. **"만들기"** 클릭
4. 목록에 서버가 추가되면 클릭하여 선택

---

### 2단계: 파일 업로드

서버를 선택한 후 파일을 추가합니다.

**로컬 파일:**
- 파일을 화면 왼쪽 영역으로 **드래그 앤 드롭**
- 또는 업로드 버튼 클릭 → 파일 선택

**Google Drive에서:**
1. 왼쪽 패널의 **"Google Drive"** 버튼 클릭 (Drive 연결 필요)
2. 파일 탐색기에서 원하는 파일 선택 (파일명 검색 가능)
3. **"업로드"** 클릭

**Notion에서:**
1. 왼쪽 패널의 **"Notion"** 버튼 클릭 (Notion 키 설정 필요)
2. 검색창에서 페이지 검색 또는 목록에서 선택
   - **전체 선택**: 검색창 옆 "전체" 체크박스
   - **데이터베이스 선택**: 데이터베이스 체크 시 하위 페이지 전체 자동 선택
3. **"업로드"** 클릭

파일 상태가 **"준비됨"** 이 될 때까지 기다립니다 (파일 크기에 따라 수초~수십초 소요).

---

### 3단계: 질문하기

1. 파일이 준비된 서버를 선택합니다.
2. 화면 하단 입력창에 질문을 입력합니다.
3. **Enter** 또는 전송 버튼 클릭
4. Gemini AI가 업로드된 파일 내용을 기반으로 답변합니다.

**질문 예시:**
- `3분기 매출 현황을 요약해줘`
- `휴가 신청 절차가 어떻게 되나요?`
- `이 계약서에서 위약금 조항을 찾아줘`
- `A 제품과 B 제품의 차이점을 비교해줘`

> 업로드된 파일에 없는 내용은 답변하지 않습니다. Gemini가 추측하지 않고 문서 기반으로만 답변하도록 설계되어 있습니다.

## 자주 묻는 질문 (FAQ)

**Q. 서버를 삭제하면 파일도 모두 삭제되나요?**

네. 서버 삭제 시 해당 서버에 업로드된 모든 파일이 Gemini 서비스에서도 함께 삭제됩니다. 원본 파일은 영향 없습니다.

**Q. 파일 상태가 계속 "업로드 중"으로 표시됩니다.**

Gemini API 처리 지연이 발생할 수 있습니다. 1~2분 기다린 후 서버 동기화 버튼(🔄)을 클릭하세요. 그래도 해결되지 않으면 해당 파일을 삭제 후 다시 업로드하세요.

**Q. Google Drive 연결 팝업이 열리지 않습니다.**

브라우저의 팝업 차단 설정을 확인하세요. `http://localhost:8000` 주소에 대한 팝업을 허용해야 합니다.

**Q. Notion 페이지가 목록에 나타나지 않습니다.**

Notion에서 해당 페이지에 Integration을 연결했는지 확인하세요. 페이지 우측 상단 `···` → **연결** → 생성한 Integration 추가가 필요합니다.

**Q. 앱을 재시작해도 이전에 업로드한 파일이 유지되나요?**

네. 파일 정보는 `sessions.db`에 저장되고, 실제 파일은 Gemini 서버에 저장되므로 앱을 재시작해도 목록이 유지됩니다.

**Q. 여러 명이 동시에 사용할 수 있나요?**

네. 같은 서버(파일스토어)를 여러 명이 동시에 사용할 수 있습니다. Google Drive는 브라우저별로 각자 계정을 연결합니다.

## 고급 설정

### .env 파일로 환경변수 설정

`.env.example` 을 복사하여 `.env` 로 저장한 후 편집하면 API 키를 파일로 고정 관리할 수 있습니다. 앱 UI에서 입력한 키는 내부 DB에 저장되므로 `.env` 파일은 선택 사항입니다.

```env
# Gemini API 키 (.env 설정이 UI 입력보다 우선 적용됨)
GOOGLE_API_KEY=AIzaxxxxxxxxxxxxxxxx

# Notion API 키
NOTION_API_KEY=secret_xxxxxxxxxxxx

# Google Drive OAuth 클라이언트 시크릿 파일 경로 (기본: 프로젝트 루트의 client_secret.json)
GOOGLE_CLIENT_SECRET_FILE=client_secret.json
```

### 포트 변경

`app.py` 마지막 줄의 `port` 값을 수정합니다.

```python
uvicorn.run(app, host="0.0.0.0", port=8000)  # port 값 변경
```
